3. kotlin库工程,类似于java library module.
	3.1 运行异常：Java.lang.NoClassDefFoundError: Failed resolution of: Lkotlin/jvm/internal/Intrinsics
		
		https://blog.csdn.net/zhendless/article/details/76152420 
		https://stackoverflow.com/questions/60059357/java-lang-noclassdeffounderror-kotlin-jvm-internal-intrinsics-in-spring-boot-ap
		
		使用库文件：github加速地址。
		https://codechina.csdn.net/mirrors/mapzen/on-the-road_android?utm_source=csdn_github_accelerator
		添加依赖：
		'com.mapzen:on-the-road:1.3.1'
		
		添加后。运行成功。再删除该库也没有任何影响。
		分析：可能不是这个原因。
		
		发现依赖库kotlin-stdlib有警告提示：
		implementation "org.jetbrains.kotlin:kotlin-stdlib-jdk8:$kotlin_version"
			提示为：Plugin version (1.3.50) is not the same as library version (idk8-1.3.50) less...(Ctrl+F1)
			有警告也不影响。
			修改为以下配置就没有警告了：implementation "org.jetbrains.kotlin:kotlin-stdlib:$kotlin_version"
		Kotlin not configured
		分析原因为：没有配置kotlin-stdlib依赖库。
			https://blog.csdn.net/qq_33143459/article/details/103989807
