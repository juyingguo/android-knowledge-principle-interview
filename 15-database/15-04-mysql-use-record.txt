					修改ImagePath,把值改成（如下，根据自己电脑路径填写）：
	
					"D:\juying\java2\mysql-8.0.28-winx64\bin\mysqld" mysqld --defaults-file=D:\juying\java2\mysql-8.0.28-winx64\my-mysql-8.0.28.ini  MYSQL8028
					
					"D:\softLoad\mysql_sql57\mysql-5.7.35-winx64\bin\mysqld" mysqld --defaults-file=D:\juying\java2\mysql-5.7.37-win32\my-mysql-5.7.37.ini MYSQL7
					
					[以上是错误的]。
					{参考已有的mysql配置（mysql-5.5）：
						"C:\Program Files\MySQL\MySQL Server 5.5\bin\mysqld" --defaults-file="C:\Program Files\MySQL\MySQL Server 5.5\my.ini" MySQL
						
						{配置mysql5.7,mysql8.0.28多了mysqld，--defaults-file后面的配置没有加双引号}
						{正确如下：
							"D:\softs-install\mysql-5.7.37-winx64\bin\mysqld" --defaults-file="D:\softs-install\mysql-5.7.37-winx64\my-mysql-5.7.37.ini" MYSQL7
						}
					}
				a3, 登录mysql（命令行输入mysql -uroot -P3307 -p +刚生成的那个root的初始密码登录，-P后面是端口）

						https://blog.csdn.net/m0_67402026/article/details/123786803
						{
							a3.2.1多种方法：
							f1:set password for root@localhost = password('root');#mysql5.7可以使用，mysql8.0.28使用这个方法报错：
							
							a3.2.2 用UPDATE直接编辑user表
							use mysql提示：
							ERROR 1820 (HY000): You must reset your password using ALTER USER statement before executing this statement.
							
							{解决办法：> alter user 'root'@'localhost' identified by '123456';}修改成功.
							{参考播客：https://blog.csdn.net/weixin_41783910/article/details/119569995}
						}
						{
							密码修改成功后。
							Navicat 连接 mysql，连接时出现问题：
							1251-Client does not support authentication protocol requested by server;consider upgrading MySQL client;
							{参考：【MySql】Navicat 连接数据库出现1251 - Client does not support authentication protocol ...... 问题的解决方法

									alter user 'root'@'localhost' identified with mysql_native_password by 'password';
									
									flush privileges;
