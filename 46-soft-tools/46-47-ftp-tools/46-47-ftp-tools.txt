	3.3 Xshell、Xftp连接虚拟机
		https://blog.csdn.net/qq_52720527/article/details/125896786
		
		第一次连接会出现问题，一直连接不上很正常，虚拟机虚拟机中为配置ssh服务
		
		Xshell或Xftp连接虚拟机，都需要虚拟机提前配置启动ssh服务。
		
		连接配置：
			服务器类型：SSH2, OpenSSH_8.2p1 Ubuntu-4ubuntu0.5
		第一次连接会出现问题，一直连接不上很正常，虚拟机虚拟机中为配置ssh服务，在终端输入一下命令：

		ps -e | grep ssh
		1
		查看反馈是否有sshd，下面这种就是未启动ssh的情况

		查看是否有安装了ssh，

		sudo service sshd start
		1
		如果出现，下面这句话就是未安装

		Failed to start sshd.service: Unit sshd.service not found.
		1
		下面我们用指令安装ssh，在终端输入以下指令。

		sudo apt install openssh-server
